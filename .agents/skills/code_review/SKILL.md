# Skill: Code Review

## Purpose

Review code like a senior engineer and provide actionable feedback.

---

## When to Use

* After code generation
* Before final output
* When validating implementation quality

---

## Responsibilities

1. Analyze code structure and readability
2. Check naming conventions
3. Detect bugs or logical issues
4. Identify performance concerns
5. Suggest improvements

---

## Guidelines

* Do NOT rewrite entire code
* Provide concise and practical suggestions
* Focus on real issues, avoid over-analysis

---

## Output Format

### Summary

Brief quality overview

### 🔴 Critical Issues

* Bugs, crashes, major problems

### 🟡 Improvements

* Structure, naming, optimization

### 🟢 Suggestions

* Optional enhancements
